## 1과목 - 소프트웨어 설계

**1. 정형 기술 검토(FTR)의 지침 사항으로 가장 옳지 않은 것은?**
① 제품의 검토에만 집중한다.
② 문제 영역을 명확히 표현한다.
③ 참가자의 수를 제한하고 사전 준비를 강요한다.
④ **논쟁이나 반박을 제한하지 않는다. ← 정답**

> **해설:** FTR에서는 논쟁과 반박을 제한해야 한다. 검토 회의가 산만해지는 것을 방지하고, 제품의 결함 발견에만 집중하기 위함이다.

**2. 유스케이스 다이어그램의 구성요소가 아닌 것은?**
① 유스케이스(Usecase)
② 액터(Actor)
③ 시스템(System)
④ **클래스(Class) ← 정답**

> **해설:** 유스케이스 다이어그램의 구성요소는 유스케이스, 액터, 시스템(경계), 관계(연관/포함/확장/일반화)이다. 클래스는 클래스 다이어그램의 구성요소이다.

**3. User Interface 설계 시 오류 메시지나 경고에 관한 지침으로 가장 거리가 먼 것은?**
① 메시지는 이해하기 쉬워야 한다.
② 오류로부터 회복을 위한 구체적인 설명이 제공되어야 한다.
③ 오류로 인해 발생할 수 있는 부정적인 내용을 적극적으로 사용자들에게 알려야 한다.
④ **소리나 색의 사용을 줄이고 텍스트로만 전달하도록 한다. ← 정답**

> **해설:** 오류 메시지는 소리, 색상 등 다양한 감각 요소를 활용하여 사용자에게 효과적으로 전달해야 한다. 텍스트만으로 제한하는 것은 올바른 지침이 아니다.

**4. 파이프 필터 형태의 소프트웨어 아키텍처에 대한 설명으로 옳은 것은?**
① 노드와 간선으로 구성된다.
② **서브 시스템이 입력 데이터를 받아 처리하고, 결과를 다음 서브 시스템으로 넘겨주는 과정을 반복한다. ← 정답**
③ 계층 모델이라고도 한다.
④ 3개의 서브 시스템(모델, 뷰, 제어)으로 구성되어 있다.

> **해설:** 파이프-필터 아키텍처는 데이터 흐름(Stream) 중심으로, 각 필터(서브시스템)가 입력을 받아 처리 후 다음 필터로 전달한다. ①은 그래프, ③은 레이어(계층), ④는 MVC 패턴 설명이다.

**5. CASE가 갖고 있는 주요 기능이 아닌 것은?**
① 그래픽 지원
② 소프트웨어 생명주기 전 단계의 연결
③ **언어 번역 ← 정답**
④ 다양한 소프트웨어 개발 모형 지원

> **해설:** CASE(Computer Aided Software Engineering)의 주요 기능은 그래픽 지원, 생명주기 전 단계 연결, 다양한 개발 모형 지원, 문서화 자동화 등이다. 언어 번역은 컴파일러의 기능이다.

**6. 다음에서 설명하는 응집도의 유형은?**
> 동일한 입력과 출력을 사용하여 다른 기능을 수행하는 활동들이 모여있을 경우의 응집도

① 논리적 응집도
② 절차적 응집도
③ 시간적 응집도
④ **교환적 응집도 ← 정답**

> **해설:** 교환적(Communication) 응집도는 동일한 입력과 출력을 사용하여 서로 다른 기능을 수행하는 요소들이 하나의 모듈에 모여있는 경우이다. 응집도 순서(낮→높): 우연적-논리적-시간적-절차적-교환적-순차적-기능적.

**7. 모듈 내부 요소 간의 관련성을 무엇이라고 하는가?**
① **cohesion ← 정답**
② coupling
③ method
④ chaining

> **해설:** Cohesion(응집도)은 모듈 내부 요소 간의 관련성, Coupling(결합도)은 모듈 간의 상호 의존도를 의미한다. 좋은 설계는 높은 응집도, 낮은 결합도를 가진다.

**8. 애자일(Agile) 프로세스 모델에 대한 설명으로 틀린 것은?**
① **변화에 대한 대응보다는 자세한 계획을 중심으로 소프트웨어를 개발한다. ← 정답**
② 프로세스와 도구 중심이 아닌 개개인과의 상호 소통을 통해 의견을 수렴한다.
③ 협상과 계약보다는 고객과의 협력을 중시한다.
④ 문서 중심이 아닌, 실행 가능한 소프트웨어를 중시한다.

> **해설:** 애자일 선언문의 핵심은 "계획을 따르기보다 변화에 대응하기"이다. ①은 전통적 폭포수 모델의 특징이며, 애자일은 변화에 유연하게 대응하는 것을 중시한다.

**9. 다음 중 객체에게 어떤 행위를 하도록 지시하는 방법은?**
① Object
② Method
③ **Message ← 정답**
④ Instance

> **해설:** 객체지향에서 Message는 객체 간 상호작용 수단으로, 다른 객체에게 행위를 요청하는 방법이다. Method는 메시지를 받았을 때 실행되는 구체적인 연산이다.

**10. 다음에서 설명하는 디자인 패턴의 유형은?**
> 각 클래스 데이터 구조로부터 처리 기능을 분리하여 별도의 클래스를 만들어 놓고, 해당 클래스의 메서드가 각 클래스를 돌아다니며 특정 작업을 수행하도록 만드는 패턴

① **Visitor ← 정답**
② State
③ Command
④ Strategy

> **해설:** Visitor 패턴은 데이터 구조와 처리 기능을 분리하여, 방문자(Visitor) 클래스가 각 요소를 순회하며 작업을 수행한다. 새로운 연산을 추가할 때 기존 클래스를 수정하지 않아도 된다.

**11. (ai복원) 다음 중 간접 연계 방식의 특징으로 옳지 않은 것은?**
① 연계 절차가 복잡하다.
② 성능 저하 가능성이 있다.
③ 테스트 및 검증 기간이 길어질 수 있다.
④ **시스템 간 실시간 처리에 가장 적합하다. ← 정답**

> **해설:** 간접 연계는 중간 매개체(EAI, ESB 등)를 통해 데이터를 전달하므로 실시간 처리보다는 비동기·배치 처리에 적합하다. 실시간 처리에 적합한 것은 직접 연계 방식이다.

**12. 애자일(Agile) 기법 중 스크럼(Scrum)과 관련된 용어에 대한 설명이 틀린 것은?**
① 스크럼 마스터는 스크럼 프로세스를 따르고, 팀이 스크럼을 효과적으로 활용할 수 있도록 보장하는 역할 등을 맡는다.
② 제품 백로그는 스크럼 팀이 해결해야 하는 목록으로 소프트웨어 요구사항, 아키텍처 정의 등이 포함될 수 있다.
③ **스프린트는 하나의 완성된 최종 결과물을 만들기 위한 주기로 3달 이상의 장기간으로 결정된다. ← 정답**
④ 속도(Velocity)는 한 번의 스프린트에서 한 팀이 어느 정도의 제품 백로그를 감당할 수 있는지에 대한 추정치로 볼 수 있다.

> **해설:** 스프린트는 보통 2~4주의 짧은 반복 주기이다. 3달 이상의 장기간이 아니라 짧은 주기로 빠르게 결과물을 만들어내는 것이 스크럼의 핵심이다.

**13. (ai복원) 다음 중 현행 시스템 파악 절차에 해당하지 않는 것은?**
① 시스템 구성 파악
② 기능 파악
③ 인터페이스 현황 파악
④ 네트워크 구성 파악
⑤ **사용자 요구사항 명세 작성 ← 정답**

> **해설:** 현행 시스템 파악 단계는 현재 운영 중인 시스템 분석 단계이다.
> 요구사항 명세 작성은 차세대 시스템 설계 단계에 해당한다.

**14. (ai복원) 소프트웨어가 사용자의 요구사항을 올바르게 충족하는지를 확인하는 개념은?**
① 신뢰성
② 효율성
③ 완전성
④ **유효성(Validity) ← 정답**
⑤ 무결성

> **해설:** Verification: 제품을 올바르게 만들었는가 / Validation: 올바른 제품을 만들었는가.
> 사용자 요구를 충족하는지 확인하는 것은 Validation(유효성)이다.

**15. (ai복원) 다음 중 동적(행위) 다이어그램이 아닌 것은?**
① 시퀀스 다이어그램
② 활동 다이어그램
③ 상태 다이어그램
④ **클래스 다이어그램 ← 정답**
⑤ 커뮤니케이션 다이어그램

> **해설:** 동적 다이어그램: 시퀀스, 활동, 상태, 커뮤니케이션.
> 정적(구조) 다이어그램: 클래스, 객체, 컴포넌트, 배치 등.
> 클래스 다이어그램은 구조 다이어그램이다.

**16. (ai복원) 캡슐화(Encapsulation)의 효과로 가장 적절한 것은?**
① 코드 재사용성 증가
② 객체 간 결합도 증가
③ **내부 구현 세부사항 보호 ← 정답**
④ 실행 속도 향상
⑤ 클래스 수 감소

> **해설:** 캡슐화의 핵심은 정보 은닉(Information Hiding)이다.
> 외부에서 객체 내부 데이터를 직접 접근하지 못하게 하여 결합도를 낮추고 유지보수를 용이하게 한다.

**17. 다음 중 행위(Behavioral) 패턴이 아닌 것은?**
① Observer
② Strategy
③ Command
④ **Proxy ← 정답**
⑤ State

> **해설:** 생성 패턴: Singleton, Factory, Builder 등.
> 구조 패턴: Proxy, Adapter, Bridge 등.
> 행위 패턴: Observer, Strategy, Command, State 등.
> Proxy는 구조 패턴이다.

**18. CPM(Critical Path Method)에 대한 설명으로 가장 옳지 않은 것은?**
① 각 작업의 수행 시간이 고정되어 있다고 가정하는 확정적(Deterministic) 모델이다.
② 프로젝트를 완료하기 위한 최소 기간을 결정하는 최장 경로(Critical Path)를 산출한다.
③ 모든 자원 제약사항을 배제한 상태로 프로젝트의 시작과 끝을 나타내는 노드와 노드 간의 연결을 통해 공정을 계산하기 위한 액티비티 표기법이다.
④ **낙관치, 비관치 등 통계적 기법을 사용하여 프로젝트 일정의 불확실성을 시뮬레이션하는 데 유리하다. ← 정답**

> **해설:** CPM은 확정적 모델로 하나의 고정된 작업 시간을 사용한다. 낙관치·비관치·최빈치 등 3점 추정을 사용하는 것은 PERT(Program Evaluation and Review Technique)이다.

**19. 다음 중 애자일 기법이 아닌 것은?**
① XP
② Crystal
③ Scrum
④ **SPICE ← 정답**

> **해설:** SPICE(Software Process Improvement and Capability dEtermination)는 소프트웨어 프로세스 평가를 위한 국제 표준(ISO/IEC 15504)이다. 애자일 기법이 아니다.

**20. 다음에서 설명하는 사용자 인터페이스의 설계 원칙은?**
> 정확하고 완벽하게 사용자의 목표가 달성될 수 있도록 제작

① 직관성(Intuitiveness)
② **유효성(Effectiveness) ← 정답**
③ 학습성(Learnability)
④ 유연성(Flexibility)

> **해설:** UI 설계 원칙 — 직관성: 누구나 쉽게 이해, 유효성: 사용자 목표를 정확하고 완벽하게 달성, 학습성: 쉽게 배울 수 있음, 유연성: 다양한 요구사항 수용.

## 2과목 - 소프트웨어 개발

**21. 소프트웨어 재공학의 주요 활동 중 기존 소프트웨어 시스템을 새로운 기술 또는 하드웨어 환경에서 사용할 수 있도록 변환하는 작업은?**
① Analysis
② **Migration ← 정답**
③ Restructuring
④ Reverse Engineering

> **해설:** Migration(이전)은 기존 시스템을 새로운 환경으로 옮기는 것이다. Restructuring(재구성)은 코드 구조 개선, Reverse Engineering(역공학)은 설계 정보 추출, Analysis(분석)은 시스템 이해 단계이다.

**22. 알파, 베타 테스트와 가장 밀접한 연관이 있는 테스트 단계는?**
① 단위 테스트
② **인수 테스트 ← 정답**
③ 통합 테스트
④ 시스템 테스트

> **해설:** 알파 테스트는 개발자 환경에서 사용자가 수행, 베타 테스트는 실제 사용 환경에서 사용자가 수행하며, 둘 다 인수 테스트(Acceptance Test)의 유형이다.

**23. 비선형 자료 구조에 해당하는 것은?**
① 큐(Queue)
② **그래프(Graph) ← 정답**
③ 데크(Deque)
④ 스택(Stack)

> **해설:** 선형 자료구조: 배열, 연결 리스트, 스택, 큐, 데크. 비선형 자료구조: 트리, 그래프. 그래프는 노드와 간선으로 구성된 비선형 구조이다.

**24. 스택(Stack)에 대한 옳은 내용으로만 나열된 것은?**
> ㉠ FIFO 방식으로 처리된다.
> ㉡ 순서 리스트의 뒤(Rear)에서 노드가 삽입되며, 앞(Front)에서 노드가 제거된다.
> ㉢ 선형 리스트의 양쪽 끝에서 삽입과 삭제가 모두 가능한 자료 구조이다.
> ㉣ 인터럽트 처리, 서브루틴 호출 작업 등에 응용된다.

① ㉠, ㉡
② ㉡, ㉢
③ **㉣ ← 정답**
④ ㉠, ㉡, ㉢, ㉣

> **해설:** ㉠은 큐(FIFO), ㉡도 큐 설명, ㉢은 데크(Deque) 설명이다. 스택은 LIFO 방식이며 인터럽트 처리, 서브루틴 호출, 후위 표기법 연산 등에 사용된다.
